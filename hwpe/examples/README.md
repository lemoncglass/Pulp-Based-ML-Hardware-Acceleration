# PULP Example Applications

Working copies of PULP-SDK test programs, used as the basis for RedMulE
accelerator experiments.

## Contents

| Directory | Source | Description |
|-----------|--------|-------------|
| `hello/` | `pulp-sdk/tests/hello/` | Minimal "Hello World" — useful for verifying the toolchain + GVSoC work. |
| `redmule/gemm/` | `pulp-sdk/tests/redmule/gemm/` | FP16 GEMM test for the RedMulE accelerator. |
| `redmule/gemm/inc/test_header_generation/` | *our code* | Python scripts for generating custom FP16 test vectors. |

> **Note:** All files outside `test_header_generation/` are unmodified copies
> from [pulp-platform/pulp-sdk](https://github.com/pulp-platform/pulp-sdk).
> The `og files for reference/` folder contains the original header data that
> shipped with the upstream test.

---

## Quick Start

```bash
# From the repo root — make sure environment is loaded
source setup_env.sh

# Run hello world
cd hwpe/examples/hello && make clean all run

# Run RedMulE GEMM (requires --target-property=chip/cluster/redmule=True)
cd hwpe/examples/redmule/gemm && make clean all run
```

## Generating Custom FP16 Test Vectors

Two options:

### 1. Deterministic vectors (`gen_fp16_test.py`)

Produces small sequential floats (easy to debug by hand):

```bash
cd redmule/gemm/inc/test_header_generation
python3 gen_fp16_test.py          # writes headers into current dir
cp *.h ../                        # copy into the include path
```

### 2. Official golden model (random + bit-true FMA)

Uses the ETH Zurich / UniBO [redmule-golden-model](https://github.com/yvantor/redmule-golden-model):

```bash
cd redmule/gemm/inc/test_header_generation

# Clone if not already present (setup.sh does this automatically)
git clone https://github.com/yvantor/redmule-golden-model.git

# Install Python dependencies (numpy + torch) however you prefer:
#   pip install --user numpy torch
#   — or use a venv / conda / system install

# Generate the raw golden-model headers
cd redmule-golden-model/FP16
IncludeDir=../common python3 gemm.py \
    --m_size 48 --n_size 64 --k_size 48 \
    --inc_dir ../../golden_output \
    --txt_dir ../../golden_output_txt
cd ../..

# Convert to #define macro format (required by our older redmule_test.c)
python3 convert_golden.py --input golden_output --output ..
```

> ⚠️ The golden model cleans `--inc_dir` before writing. Always use a
> separate flat directory (not `inc/` directly).

> **Format note:** The upstream golden model outputs headers using
> variable declarations (`uint16_t x_inp[3072] = {...};`), but our
> version of `redmule_test.c` expects `#define X {...}` macros.
> `convert_golden.py` handles this translation automatically. It also
> patches `tensor_dim.h` to add the `#include "../archi_redmule.h"`
> that the older SDK needs.

## RedMulE Upstream Reference

The full RedMulE README (hardware architecture, programming model, ISA extensions,
etc.) is downloaded by `setup.sh` into `redmule/RedMulE_README.md`. If it's missing:

```bash
curl -fsSL https://raw.githubusercontent.com/pulp-platform/redmule/main/README.md \
  -o redmule/RedMulE_README.md
```
